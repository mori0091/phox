# トレイト解決と `let` / `*let` の違い

Phox の `+` などの基本演算子は、対応するトレイト（例：`Add a`）によって提供されます。  
複数の型に対して `Add` の実装が存在する場合、**どの実装を使うかはスコープと束縛によって決まります**。

## `let` は単相（monomorphic）
`let` で定義される関数は **その場で型が 1 つに決まっている必要があります**。  
そのため、次のような定義は曖昧でエラーになります。

```
let add = \a.\b. a + b;
// => ambiguous variable `(+)`
```

`a + b` の時点で `a` の型が決まっていないため、  
`Add Int` と `Add u8` のどちらを使うべきか判断できません。

> [NOTE!]  
> 文脈によって `let` は多相（polymorphic）にも成り得ます。  
> 例えば、モジュールトップレベルで `let` で定義された次のような関数は多相関数になります。  
> ```
> // id : ∀ a. a -> a
> let id = \x. x;
> ```
>
> 前述の `add` のケースでは、`(+)` が多重定義された複数の候補がある単相関数であり、
> 多相関数ではない（型が限定されている関数である）ため、曖昧エラーになります。

> [NOTE!]  
> `let` が多相化されるのは「その定義が曖昧性を含まない場合」に限られます。  
> 例えば `let id = \x. x` は、`x` に対する制約が何もないため多相化されます。  
>
> 一方、`a + b` のように **トレイト解決が必要な式** を含む場合、  
> その場でどのトレイト実装を使うか決められなければ `let` は多相化されません。  
>
> つまり、`let` が多相化されるかどうかは  
> **「型推論がその場で一意に決まるかどうか」**  
> によって決まります。

## `*let` は多相（polymorphic）
`*let` は **型制約つき多相関数** を定義できます。  
演算子の実装は **使用される文脈に応じて、コンパイル時に静的に解決されます**。  
（辞書渡しではなく、型ごとに実装が静的に焼きこまれるイメージです）

```
*let add = @{Add a}.(+);

add 2 3       // Add Int が選ばれる
add 2u8 3u8   // Add u8 が選ばれる
```

> `*let` は「曖昧でも定義できる」が、「呼び出し時に曖昧ならエラー」になります。

## リテラルによる型決定
リテラルが含まれると、その部分で型が確定するため曖昧性が解消されます。

```
let add = \a.\b. 0 + a + b;
// 0 が Int のため、Add Int が選ばれる
```

## ローカルスコープで演算子の実装を固定する
ブロック `{ ... }` 内でトレイトレコードを束縛すると、  
そのスコープ内では特定の実装だけが使われます。

```
let add = {
  let @{(+)} = @{Add Int};
  \a.\b. a + b
};

add 2 3   // => 5: Int
```

これは次のように書いても同じ意味です。

```
let add = {
  let (+) = @{Add Int}.(+);
  \a.\b. a + b
};
```

さらに短く書くと、次のように **演算子そのものを束縛**できます。

```
let add = @{Add Int}.(+);

add 2 3   // => 5: Int
```

> この方法は「スコープ内で (+) の候補が 1 つだけになる」ため、曖昧性が解消されます。

## まとめ
- `let` は単相：その場で impl を 1 つに決める必要がある  
- `*let` は多相：impl は使用される文脈で静的に決まる  
- リテラルは型を確定させる  
- `{ let @{(+)} = @{Add Int}; ... }` で局所的に impl を固定できる  
- トレイトレコードは辞書ではなく **名前空間の束縛**  
- 曖昧性が残る場合、Phox は常にエラーにする（暗黙の default は存在しない）

> トレイトレコードは「辞書」ではなく、**レコード値**です。  
> パターンマッチと組み合せる（ex. `let @{(+)} = @{Add Int};`）ことで  
> **「このスコープではこの実装を使う」という名前空間の切り替え**  
> として機能します。
